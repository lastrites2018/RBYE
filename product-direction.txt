RBYE 제품 방향 정리
==================

■ 핵심 정체성

  "채용 공고의 경향성을 빠르게 파악하는 도구"

  - 구직 지원 플랫폼이 아님 (잡포털 X)
  - 공고 텍스트의 패턴을 읽는 분석 도구
  - 타겟: "시장이 뭘 원하는지 알고 싶은" 개발자/PM


■ 운영 원칙: 무료·제로코스트

  - 유료화 계획 없음. 무료 인프라 범위 내에서 유지
  - 호스팅: Vercel 무료 플랜
  - 크롤링: 로컬에서 수동 실행 (서버 비용 0)
  - 분석: 로컬 Node.js 스크립트 (analyze.js)
  - DB 없음: JSON 파일 기반 (json-server)
  - 이 구조에서 벗어나는 기능은 도입하지 않음


■ 핵심 가치

  1. 속도 — 공고를 빠르게 훑을 수 있는 구조
  2. 경향성 — 연차별/기술별 시장 수요 패턴 시각화
  3. 변화 감지 — 시간에 따른 기술 트렌드 변화 추적


■ 기능 우선순위

  [핵심] 직접적으로 경향성 파악에 기여
    - 공고 탐색: 연차별 필터 + 키워드 검색으로 빠른 훑기
    - 기술 통계: 카테고리별 TOP 키워드 빈도
    - 스킬 로드맵 (탐색 모드): 연차 전환하며 요구 기술 변화 비교
    - 연차별 기술 비교 뷰: 같은 기술이 연차에 따라 어떻게 변하는지 (TODO)
    - 시계열 트렌드: 로컬 스냅샷 축적 → 정적 JSON으로 배포 (TODO)

  [보조] 유용하지만 부차적
    - 나의 스킬 체크: 경향 파악 후 "나는 어디쯤?" 확인 (localStorage, 서버 불필요)
    - 추천 기술: 보유 스킬 기반 학습 우선순위 제안

  [유지] 있어도 되지만 투자 우선순위 낮음
    - 회사 정보 더 보기 (잡플래닛/사람인 링크)
    - 공고 원문 링크


■ 하지 않을 것

  - 유료 API, 외부 DB, 서버리스 함수 등 비용 발생 요소
  - 자동 크롤링/스케줄링 (서버 필요)
  - 이력서 관리, 지원 추적 등 구직 플로우
  - 공고 카드에 과도한 UI 추가 (pill, 태그 등) — 빠른 훑기 방해
  - 개별 회사 분석 — 회사가 아닌 "시장 전체"가 대상


■ 아키텍처 제약

  데이터 흐름: 로컬 크롤링 → analyze.js → stats.json → git push → Vercel 배포
  런타임 비용: 0원 (정적 JSON + Vercel 무료)
  모든 연산은 빌드타임 또는 클라이언트에서 처리
  시계열도 history/*.json을 로컬에서 쌓고 정적으로 배포


■ 데이터 파이프라인

  크롤링 실행 시 순서:
    1) 크롤링 실행 (공고 JSON 갱신)
    2) node discover-keywords.js     → 새 키워드 후보 생성 (keyword-candidates.json)
    3) keyword-candidates.json에서 approved: true + slot 지정 (수동)
    4) node discover-keywords.js --apply  → 승인 키워드를 tech-dictionary.json에 반영
    5) node analyze.js               → stats.json 생성 + 시계열 스냅샷 저장

  키워드 발견 기준:
    - 10회 이상 언급된 새 키워드만 후보로 제안
    - 완전 자동 추가는 하지 않음 (노이즈 유입 방지)
    - 승인 시 slot(기본장착/주무기/보조장비/전투도구/고급장비/AI활용) 지정 필수
